### 🧭 **Working Style & Approach**

I approach engineering problems similarly to how quantitative researchers approach markets: through **first-principles thinking, experimentation, and iteration**.

**1. Deconstruct the System**

Complex problems are broken into fundamental mathematical or computational components.

Examples:

- Swarm autonomy → graph theory + distributed consensus  
- Portfolio optimization → convex optimization + stochastic processes  
- Machine learning → linear algebra + gradient-based optimization  

**2. Build From First Principles**

Before relying on high-level frameworks, I prefer implementing core mechanisms myself.

Examples:

- Neural networks from scratch  
- Data structure visualizers and algorithm simulations  
- Financial models implemented manually  

This builds deeper **system intuition and mathematical understanding**.

**3. Engineer for Performance**

Once the core idea works, the focus shifts to **engineering robust systems**.

Typical improvements include:

- scalable architecture  
- framework integration (TensorFlow, Node, etc.)  
- performance optimization and maintainability  

**4. Iterate Like a Researcher**

Most projects follow an experimentation cycle:
Hypothesis → Prototype → Experiment → Analyze → Improve

**5. Document and Share**

Knowledge compounds when shared.

I prioritize:

- open-source libraries  
- clear documentation  
- visual tools for understanding complex systems
